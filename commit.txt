Add auto-startup feature for model loading and hotkey registration

Implement optional auto-startup functionality that allows users to automatically
load the Whisper model and register the global hotkey when the application starts.
This eliminates the need for manual initialization steps each time the app launches.

- Add settings persistence for auto_load_model and auto_register_hotkey
- Add tests for new configuration defaults and settings persistence

Both settings default to False, maintaining existing behavior. When enabled:
- Auto-load runs in background thread to prevent GUI freeze
- Auto-register only triggers if model loads successfully
- Errors are handled gracefully with user notifications
- App remains usable if auto-startup fails
